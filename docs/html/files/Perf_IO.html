Perf_IO_fcts.perm_init_config_reader_yml(input_file_path)[source]¶

Parameters from a configuration file are stored in ‘configs’ - a nested Python dictionary.

Read more about “YAML Ain’t Markup Language” configuration files here: https://yaml.org/. This function opens the config_permeability_initialiser.yaml file in readmode. Using yaml.load, this is read into ‘configs’, a nested Python dictionary. Using yaml.safeloader makes sure only simple datatypes are loaded and that code does not execute arbitrarily. A Python dictionary stores key-value pairs, such as ‘e_ref : [0, 0, 1]’. A nested dictionary like that stored in ‘configs’ is a dictionary within a dictionary. The values of K1_form and e_local are stored as NumPy arrays and K1_form is reshaped into a 3x3 tensor.

Parameters:

input_file_path (str) – A string showing the path to the relevant YAML configuration file.

Returns:

configs – This is a nested list of key-value pairs, containing the information from the YAML file.

Return type:

dict

Perf_IO_fcts.mesh_reader(mesh_file)[source]¶

Read data from the mesh.

Read about XDMF files here: https://www.xdmf.org/index.php/XDMF_Model_and_Format.

Read about DOLFINx here: https://docs.fenicsproject.org/dolfinx/main/python/generated/dolfinx.io.html#module-dolfinx.io.

This function imports mesh data from ‘clustered.xdmf’. It imports meshtag data from ‘clustered_physical_region.xdmf’, and ‘clustered_facet_region.xdmf’ files. These data are then stored as dolfinx.mesh objects in ‘mesh’, ‘subdomains’, and ‘boundaries’ variables respectively.

Parameters:

mesh_file (str) – An string showing the path to the mesh .xdmf file in config_permeability_initialiser.yaml.

Return type:

tuple[Mesh, Optional[MeshTags], Optional[MeshTags]]

Returns:

  • mesh (dolfinx.mesh.Mesh) – The clustered mesh file as a DOLFINx object.

  • subdomains (dolfinx.mesh.MeshTags) – Cell meshtags denoting volumetric regions.

  • boundaries (dolfinx.mesh.MeshTags) – Boundary meshtags denoting surface regions.

Notes

  • Meshtags are like integer tags conveniently marking different regions or boundaries of the mesh.

  • For this particular clustered.xdmf mesh:

  • 1,042,301 cells

  • 2,164,216 facets (in the whole mesh, with no repeats)

  • 159,228 facets on a boundary

  • Most of the facet meshtags are 0, indicating that the facet is internal

  • Non-zero values indicate regions on the surface

Perf_IO_fcts.initialise_permeabilities(K1_space, K2_space, mesh, permeability_folder, **kwarg)[source]¶

Initialise the three permeability tensor components K1, K2, and K3.

  1. Reads a model type flag from the input arguments (default: ‘acv’).

  2. Allocates K1 and K2 as fem.Functions in the provided function spaces.

  3. Loads K1 from file ‘K1_form.h5’ inside the specified permeability folder.

  4. Copies the values of K1 into K3 (which shares K1’s function space).

  5. If the model type is not recognised, an exception is raised.

  6. Prints the first 10 entries of K1 and K2 for debugging.

Parameters:

  • K1_space (dolfinx.fem.FunctionSpace) – Function space for K1.

  • K2_space (dolfinx.fem.FunctionSpace) – Function space for K2.

  • mesh (dolfinx.mesh.Mesh) – The mesh on which permeability tensors are defined (used implicitly via FunctionSpaces).

  • permeability_folder (str) – Directory path containing the HDF5 file “K1_form.h5” with stored permeability data.

  • **kwarg (dict) –

    Optional keyword arguments. Currently accepts:

    • model_type : str Either ‘acv’ or ‘a’. Determines how permeabilities are initialised.

Returns:

K1dolfinx.fem.Function

Primary permeability tensor field read from file.

K2dolfinx.fem.Function

Secondary permeability tensor field allocated but not filled from file.

K3dolfinx.fem.Function

A copy of K1 in the same function space.

Return type:

tuple

Raises:

Exception – If the provided model type is not ‘acv’ or ‘a’.
